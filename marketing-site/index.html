Features

Hosting workflows in one panel

Qadbak is the product at qadbak.com — a complete control panel you run on your own server, with role-based access for administrators and customers.

Domains

Create sites, subdomains, aliases, and lifecycle actions from one list.

Email

Mailboxes, forwarding, IMAP, and logs — scoped per domain for clients.

DNS & SSL

Records, Let’s Encrypt, and certificate status with clear feedback.

Databases

MariaDB databases and users managed from the panel.

Files & terminal

Native file manager and in-browser shell as the domain unix user.

Server admin

Service control, stack reload, backups, resellers, and host metrics.

Deploy

Run Qadbak on your own VPS

Point your panel hostname to the server, run the installer, and open https://your-host/login.

01

Ubuntu 22.04 VPS

Root access and DNS for your panel hostname. The installer sets up nginx, mail, and the web stack.

02

Install Qadbak

sudo bash install/qadbak-install.sh from github.com/macdirtycow/qadbak.

03

TLS & sign in

Optional Let’s Encrypt. Users open the panel at your hostname — ports 80/443 (or :11000).

Access

Two roles, one Qadbak panel

Same look and feel — different permissions on every API route.

Administrator

All domains, server settings, services, backups, resellers, and host tools.

Client

Only assigned domains: mail, DNS, SSL, databases, and files — no server-wide menus.

The name

From 2009 headlines to a real panel

Named after an infamous 2009 shell — rebuilt as a self-hosted hosting panel. This product is not affiliated with Qadbak Investments.

Around 2009, Qadbak Investments made noise in football and Formula 1 press. It became a story about buzz without backing. The brand was left behind in finance; it fits hosting control far better.
