Features

Everything in VirtualMin — easier in Qadbak

Qadbak is the product name for the panel at qadbak.com. Under the hood your data stays in VirtualMin; the UI is rebuilt for clarity and role-based access.

Domains

Create virtual servers, subdomains, aliases, and lifecycle actions from one list.

Email

Mailboxes, forwarding, spam tools, and logs — scoped per domain for clients.

DNS & SSL

Records, Let’s Encrypt, and certificate status with clear feedback.

Databases

MySQL and PostgreSQL databases and users without opening classic Webmin.

Files & terminal

File manager and SSH terminal via secure Webmin login links.

Server admin

Status, services, backups, resellers, and Webmin modules embedded when native UI is not ready yet.

Deploy

Run Qadbak on your own VPS

Point qadbak.com to your server, install the Qadbak app next to VirtualMin, and proxy HTTPS to the Node process.

01

Install VirtualMin

Ubuntu 22.04 with Remote API enabled. Qadbak uses remote.cgi on localhost.

02

Deploy Qadbak

Clone github.com/macdirtycow/qadbak, configure .env.local, build with pm2.

03

TLS at qadbak.com

Nginx reverse proxy and Let’s Encrypt. Users open the panel at https://qadbak.com/login.

Access

Two roles, one Qadbak panel

Same look and feel — different permissions on every API route.

Administrator

All domains, server settings, Webmin embeds, cloud backups, resellers, and plans.

Client

Only assigned domains: mail, DNS, SSL, databases, and files — no server-wide menus.
