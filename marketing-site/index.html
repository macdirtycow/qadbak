Why Qadbak

Four things no other panel does this well.

cPanel, Plesk and HestiaCP all run sites just fine. Qadbak is built for the moments around that — when something goes wrong, when you want to know what just happened, or when a junior admin needs guardrails.

01
Explain mode

Every action is a journal entry.

Every panel click writes a structured, append-only entry: the exact shell commands run, the files touched, the services reloaded. Click "Add mailbox" and see the actual doveadm pw, the line added to /etc/dovecot/passwd, and the reload — in plain English, with passwords redacted.

  • Append-only JSONL, sanitized, per-server
  • Filter by user, action type, status, undoability
  • Perfect for handover, audits and "what changed?"
02
Reversible

Undo is a button, not a backup restore.

Safe actions — mailboxes, DNS records, aliases — are reversed by the server with a single click. Every undo runs a real server-side dispatcher with safety checks: e.g. mailbox undo refuses if the inbox is no longer empty. No prayer required.

  • Currently undoable: mailbox.add, dns.record.add, dns.record.delete, alias.add, alias.delete
  • 24-hour TTL so cleanup is sane
  • Re-journals the undo step itself
03
Self-healing

The server flags its own problems.

Disk filling, RAM saturated, SSL cert about to expire, services that should be running but aren't — the health dashboard checks all of it in parallel and explains what's wrong in plain English, with the exact command you'd run to fix it next to a copy button.

  • Disk, memory, SSL expiry, service status
  • Severity-ranked: critical / warning / info
  • Distinguishes "not installed" from "installed but stopped"
04
Intent-based

WordPress in one click. Really.

Pick a template, fill three fields, click install. The orchestrator creates the domain, provisions the database and user, downloads WordPress, generates fresh salts, writes a real wp-config.php and hands you the admin URL — all inside a single journal entry so you can see exactly what happened.

  • Today: WordPress (more templates coming)
  • Every install is one journaled, reviewable operation
  • No vendor stack — vanilla nginx + PHP-FPM + MariaDB
